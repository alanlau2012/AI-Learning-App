# 验收清单 · acceptance-checklist

> 以 PRD 第 16 节为基础重构为可勾选清单。**视觉验收以 [design.md](../design.md) 为准**。
> 来源标注：`PRD` = 产品需求文档；`design` = design.md；`schema` = content-schema.md；`anim` = animation-spec.md。

## 1. 功能验收（来源：PRD §16.1）

- [ ] 用户可以浏览 6 个模块
- [ ] 用户可以进入任意知识点详情页
- [ ] 用户可以播放至少 4 类动画
- [ ] 用户可以完成诊断题并查看解析
- [ ] 用户可以标记知识点完成 / 取消完成
- [ ] 用户可以收藏 / 取消收藏知识点
- [ ] 用户刷新页面后进度不丢失（LocalStorage，带 `version` 字段）
- [ ] 用户可以搜索知识点（实时、可跳转、空结果提示）
- [ ] 用户可以查看术语索引（基础版）
- [ ] 用户可以清空学习记录
- [ ] 移动端与桌面端均可正常使用

## 2. 内容验收（来源：PRD §16.2 + schema §4）

- [ ] 56 讲信息架构完整（6 模块、模块构成 `10/10/8/16/6/6`、登记表齐全）
- [ ] 首版至少内置 12 个完整知识点
- [ ] 每个已上线知识点包含一句话定义（`definition`）
- [ ] 每个已上线知识点包含机制讲解（`mechanism`）
- [ ] 每个已上线知识点包含企业案例（`enterpriseCase`）
- [ ] 每个已上线知识点包含常见误区（`pitfalls`）
- [ ] 至少 8 个知识点包含诊断题（`diagnosticQuestion`）
- [ ] 至少 6 个知识点包含动画配置（`animation`），覆盖至少 4 类动画
- [ ] `relatedConceptIds` 全部指向已存在的知识点 id（无悬空引用）

## 3. 视觉验收（来源：design.md，**覆盖 PRD §16.3 的“华为红”方案**）

> 说明：PRD §16.3 原要求“整体白底 + 华为红作为唯一强调色”。本项目以 design.md 高保真原型为准，**改为暖纸色背景 + 蓝/绿强调**，下列为生效标准。

- [ ] 背景为温暖纸色（`--color-bg: #FAF9F6`），非纯白、非深色科技背景
- [ ] 蓝色 `#1F40D8` 仅用于主操作、当前选中、章节编号与关键强调
- [ ] 绿色 `#2E7D58` 仅用于学习进度与完成状态
- [ ] **未使用华为红作为主视觉**
- [ ] 未使用花哨渐变、霓虹/AI 光效背景
- [ ] 未使用大面积深色主题（动画机制画布的深色面积克制，外层仍为浅色）
- [ ] 边框优先于阴影，未使用强阴影卡片堆叠，未使用过度圆角（6–8px）
- [ ] 字体关系正确：标题 serif、正文 sans、序号/标签/指标 mono
- [ ] 信息密度低噪音：首页第一屏不堆满所有模块/推荐/诊断题/统计
- [ ] 详情页正文最大宽度约 760–860px，长阅读节奏舒适
- [ ] 移动端没有横向滚动
- [ ] 页面不像“AI 工程监控后台”，而像“可交互的 AI 应用工程教材”

## 4. 工程验收（来源：PRD §16.4 + architecture.md）

- [ ] TypeScript 无类型错误（`npm run typecheck`）
- [ ] ESLint 无严重错误（`npm run lint`）
- [ ] 构建成功（`npm run build`）
- [ ] 路由刷新不报错
- [ ] 组件拆分清晰，数据与组件分离（内容只在 `src/data/*`）
- [ ] 数据 schema 唯一定义在 `src/types/index.ts`，与 content-schema.md 一致
- [ ] 动画组件可复用、由配置驱动（与 animation-spec.md 一致）
- [ ] 状态管理集中于 `store/progressStore.ts`，持久化 key 为 `ai-learning-app-progress-v1`
- [ ] 视觉变量集中于 `src/styles/tokens.css`
- [ ] 无未使用的大型依赖
- [ ] README 说明如何启动项目，可静态部署

## 5. 成功标准（来源：PRD §20）

- [ ] 用户打开后能立刻理解这是 AI 应用工程学习系统
- [ ] 用户能沿 6 个模块形成完整知识地图
- [ ] 用户能通过动画看懂复杂机制
- [ ] 用户能通过诊断题训练工程判断
- [ ] 用户能记录学习进度
- [ ] 项目结构清晰，后续可扩展到 100 个知识点
- [ ] 内容、动画、题目均可通过数据配置持续扩展
