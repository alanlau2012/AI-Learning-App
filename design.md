# AI 应用工程学习 APP 设计规范

## 1. 设计方向

本项目采用 **轻量工程学习书桌** 风格。

它不是高密度控制台，也不是营销型课程页面，而是一个适合长期学习 AI 应用工程知识的安静产品界面。整体感觉应接近一本可交互的工程教材：左侧稳定目录，中间专注阅读，少量操作按钮，信息按章节自然展开。

核心关键词：

- 安静
- 专业
- 阅读友好
- 工程感
- 低信息噪音
- 可长期使用

第一屏只放当前任务最需要的信息，不把所有模块、统计、推荐、诊断题和功能入口一次性铺满。

## 2. 视觉语言

### 色彩

主界面使用温暖的纸色背景，而不是纯白或深色科技背景。

```css
:root {
  --color-bg: #FAF9F6;
  --color-surface: #FFFFFF;
  --color-surface-soft: #F2EFE7;
  --color-border: #E7E3D9;
  --color-border-strong: #DAD6CB;

  --color-title: #1A1916;
  --color-text: #34322C;
  --color-muted: #8C887E;
  --color-faint: #B4B0A6;

  --color-primary: #1F40D8;
  --color-primary-soft: #EBEEFE;
  --color-primary-border: #C9D2FB;

  --color-progress: #2E7D58;
  --color-progress-soft: #E8F2EC;
  --color-warning: #E8943A;
}
```

使用规则：

- 蓝色只用于主操作、当前选中状态、章节编号和关键强调。
- 绿色只用于学习进度和完成状态。
- 暖灰用于边框、分割线、元信息和背景层级。
- 不使用华为红作为主视觉。
- 不使用花哨渐变、大面积深色背景、霓虹科技效果或装饰性光斑。

### 字体

建议使用三类字体关系，形成“工程教材”气质。

```css
--font-sans: "IBM Plex Sans", -apple-system, BlinkMacSystemFont, "Segoe UI",
  "PingFang SC", "Microsoft YaHei", sans-serif;

--font-serif: "IBM Plex Serif", Georgia, "Times New Roman", "Noto Serif SC",
  serif;

--font-mono: "IBM Plex Mono", "SFMono-Regular", Consolas, monospace;
```

使用规则：

- 产品名、大标题、知识点标题使用 serif，形成教材感。
- 正文、导航、按钮使用 sans，保证可读性。
- 序号、进度、标签、元信息、技术标记使用 mono。
- 正文字号以 15-16px 为主，行高 1.6-1.75。
- 不使用过大的 hero 字体；页面应像学习工具，不像落地页。

## 3. 布局骨架

### 桌面端

桌面端采用固定左侧导航 + 顶部工具条 + 中间阅读画布。

```text
┌───────────────┬────────────────────────────────────┐
│ Sidebar 256px │ Top Bar                            │
│               ├────────────────────────────────────┤
│ Navigation    │                                    │
│ Modules       │ Main Reading Canvas                │
│ Progress      │ max-width: 760-1120px              │
│               │                                    │
└───────────────┴────────────────────────────────────┘
```

布局规则：

- 左侧栏宽度约 256px，固定在左侧。
- 主内容区背景使用 `--color-bg`。
- 首页主内容最大宽度约 1120px。
- 知识点详情正文最大宽度约 760-860px。
- 页面左右留白 40px 左右。
- 不使用复杂三栏控制台。
- 不把所有模块和统计同时塞进第一屏。

### 移动端

MVP 第一优先级是桌面 Web。移动端后续适配时应保留同样的阅读节奏：

- 左侧导航收起为抽屉或底部导航。
- 主内容单列。
- 首页只保留继续学习、进度、推荐路径的核心入口。
- 知识点页优先保证阅读舒适，不堆叠过多侧边信息。

## 4. 页面设计

### 首页

首页目标：让用户立刻知道这是什么产品，并能继续学习。

第一屏只放：

- 左侧导航
- 顶部搜索
- 连续学习天数
- 主标题区
- 继续学习按钮
- 简洁学习进度
- 推荐学习路径露出一小部分

首页主卡片内容建议：

```text
交互式知识系统
从模型原理
到 Agent 工厂

把 AI 应用工程的关键知识点，从“听过概念”
提升到能解释机制、能判断方案、能诊断问题、能指导落地。

[继续学习 · Token →]   查看我的学习
```

进度面板建议：

```text
0 / 56
已掌握知识点

0% 完成度   0 收藏   1 连续天数
```

不要在首页第一屏同时完整展示六大模块、今日推荐、诊断题、搜索结果、术语索引和大量统计。首页要像入口，不像报表。

### 知识点详情页

知识点详情页目标：让用户进入阅读状态。

第一屏结构：

```text
← 模型怎么跑得又快又稳

高级 · 12 分钟 · 含动画演示

KV Cache

| 一句话定义
| KV Cache 是在大模型推理中缓存历史上下文的 Key 和 Value，
| 避免每次生成新 Token 时重复计算全部历史。

01 为什么重要
KV Cache 直接影响首字时延、吞吐、显存占用和多轮对话体验。
企业 MaaS 平台如果 KV Cache 命中率低，高并发下会快速劣化。

心智模型
把 KV Cache 理解为“模型已经读过的上下文笔记”。
```

设计规则：

- 标题使用大号 serif。
- 一句话定义使用左侧 3px 蓝色竖线强调。
- 章节编号用 mono + 蓝色。
- 正文保持长阅读节奏，不拆成太多卡片。
- “心智模型”“企业案例”等可以使用浅暖灰底块，但不要堆叠阴影卡片。

### 动画演示页 / 区块

动画演示是知识点详情页中的一个章节，而不是单独的复杂控制台。

结构：

```text
03 动画演示

┌──────────────────────────────────────────────┐
│ 深色机制画布                                  │
│ input tokens -> KV Cache slots               │
│ TTFT 0.8s                                    │
└──────────────────────────────────────────────┘

[←] [播放] [→] [重置]                   3 / 5

03 写入 KV Cache
Prefill 阶段会把上下文中的 Key 和 Value 写入缓存，后续 Decode 阶段复用。
```

设计规则：

- 允许动画画布使用深色背景，但面积应克制，只作为机制演示容器。
- 深色画布外层仍然在浅色阅读页面中。
- 控制区保持简单：上一步、播放/暂停、下一步、重置、步骤计数。
- 不增加监控面板、复杂指标、右侧大说明栏。
- 每次只解释当前一步。

## 5. 导航与组件

### Sidebar

左侧栏是整个产品的稳定锚点。

内容顺序：

1. 产品名：`AI 应用工程`
2. 小标签：`50 讲`、`MVP · v1`
3. 一级导航：`首页`、`我的学习`、`搜索`
4. 六大模块列表
5. 底部总进度

六大模块显示：

```text
01 模型怎么工作             0/10
02 模型怎么跑得又快又稳     0/10
03 模型怎么变成企业平台     0/8
04 模型怎么变成 Agent       0/12
05 Agent 怎么改变软件工程   0/6
06 企业怎么治理 AI          0/10
```

规则：

- 当前导航项使用淡蓝底。
- 模块进度使用低对比 mono 文本。
- 不在侧边栏放复杂图表。

### Top Bar

顶部工具条保持轻。

内容：

- 当前 breadcrumb
- 搜索框
- 连续学习天数

规则：

- 高度约 56px。
- 背景可使用 `rgba(250, 249, 246, 0.86)` + blur。
- 搜索框宽度约 220-320px。
- 不增加通知、头像、复杂设置入口，除非后续产品明确需要。

### Buttons

主按钮：

```css
.button-primary {
  background: #1F40D8;
  color: #fff;
  border-radius: 6px;
  padding: 11px 20px;
  font-weight: 600;
}
```

次级按钮：

- 透明背景
- 蓝色文字
- 无边框或轻边框

按钮文字要短：

- `继续学习 · Token →`
- `查看我的学习`
- `播放`
- `重置`
- `完成学习`

### Cards / Panels

卡片只用于承载明确独立对象，例如首页主入口、进度面板、学习路径项。

规则：

- 边框优先于阴影。
- 圆角 6-8px。
- 不做卡片套卡片。
- 列表优先使用行分割，不要每一项都做强卡片。

## 6. 信息密度规则

最重要的调整：降低第一屏信息量。

每个页面遵循：

- 首页：一个核心动作，继续学习。
- 详情页：一个核心知识点，先读懂定义。
- 动画页：一个机制步骤，先看懂当前状态。
- 诊断题页：一个企业场景，先完成判断。

不要在同一屏同时展示：

- 六大模块完整卡片
- 今日推荐
- 复习任务
- 诊断题
- 大量学习统计
- 术语索引
- 相关知识点
- 动画播放器

这些内容可以存在，但应在页面下方或后续页面中逐步出现。

## 7. MVP 页面优先级

第一版实现建议按以下页面落地：

1. 首页
2. 模块列表页
3. 知识点详情页
4. 动画演示组件
5. 诊断题组件
6. 搜索页
7. 我的学习页

视觉优先级：

1. 先保证首页和知识点详情页气质正确。
2. 再保证动画组件清晰。
3. 最后补齐诊断题、搜索和进度统计。

## 8. 禁止项

为了保持这套风格，明确不要做：

- 不使用华为红作为主色。
- 不做大面积深色主题。
- 不做霓虹渐变和 AI 光效背景。
- 不做复杂 dashboard 式首页。
- 不在第一屏堆满所有功能。
- 不使用强阴影卡片堆叠。
- 不使用过度圆角。
- 不把诊断题、动画、模块、推荐和统计同时挤在首页第一屏。
- 不为了展示功能而牺牲阅读节奏。

## 9. 一句话判断标准

如果一个页面看起来像“AI 工程监控后台”，就太重了。

如果它看起来像“一本可以交互的 AI 应用工程教材”，就是正确方向。
