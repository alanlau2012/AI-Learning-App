# sampling：采样策略

## 一句话定义
采样策略是在模型给出下一个 token 概率分布后，决定选哪个 token 的规则，直接影响输出的确定性、多样性和风险。

## 为什么重要
同一个模型在不同 temperature、top-p、top-k、seed 和停止条件下，会表现出不同的稳定性和创造性。企业应用不是一律追求“更聪明”或“更有创意”，而是要按任务分层：财务、法务、运维诊断更重稳定和可验证，营销文案、头脑风暴可以保留多样性。采样策略出问题时，用户看到的是同问不同答、格式漂移、引用不稳定或 Agent 行动选择摇摆。

## 心智模型
采样像从候选答案里抽签，但签筒不是均匀的。模型先给每个下一个 token 一个概率，采样策略决定是总拿最高分，还是允许低概率但有可能更有创造性的选项进入。工程判断的关键不是“随机好不好”，而是任务是否允许随机。

## 机制拆解
1. 模型每一步输出的是下一个 token 的概率分布，而不是直接输出唯一答案。
2. 贪心解码总选最高概率 token，稳定但可能保守。
3. temperature 会调整概率分布的尖锐程度，越高越容易选择低概率 token。
4. top-p/top-k 会限制候选 token 集合，控制随机范围。
5. 企业系统需要把采样参数与任务类型、风险级别、评测指标和回放策略绑定。

## 企业案例
**标题：合同摘要同问不同答引发审计风险**

- 场景：某合同管理系统每月生成约 4 万份合同摘要，供法务初筛风险条款。
- 问题：同一合同重复生成摘要时，风险条款表述和排序不稳定；抽样 500 份，关键条款漏报率在不同批次间从 3% 波动到 11%。
- 分析：平台把营销文案任务的 temperature=0.8 复用到合同摘要；同时没有固定输出 schema 和引用要求，导致模型在低概率表达之间游走。
- 方案：按任务风险分层，合同摘要使用低 temperature、固定 schema、条款引用和回放评测；创意类任务保留较高多样性但不得进入审计链路。
- 结论：采样参数是产品策略和风险控制的一部分，不是随手调的模型小旋钮。

## 常见误区
1. 把 temperature 当成“智能程度”旋钮，越高越好。
2. 所有任务共用一套采样参数，忽略风险等级。
3. 只看单次输出质量，不看同输入重复生成的一致性。
4. 用采样参数掩盖提示词、检索证据或输出 schema 问题。
5. Agent 工具选择也使用过高随机性，导致行动路径不稳定。

## 诊断题
**场景**：合同摘要服务同一输入重复运行 5 次，条款排序和风险描述差异很大。业务要求摘要可审计、可复现。当前 temperature=0.9，并且没有强制引用条款编号。

**问题**：第一步最合理的处理是什么？

A. 将高风险摘要任务切到低随机采样，并固定输出 schema 与引用要求  
B. 增加候选摘要数量，让用户从 5 个版本里选择最好的  
C. 扩大 top-p，让模型覆盖更多可能表达  
D. 保持参数不变，只在页面提示“AI 结果仅供参考”

**正确答案**：A

**解析**：A 直接匹配可审计任务的稳定性要求。B 是强干扰项，多候选适合创意探索，但会增加审计负担。C 会进一步扩大随机范围。D 是免责声明，不能解决系统行为不稳定。

**排查路径**：
1. 标记任务风险等级和可复现要求。
2. 对同一输入重复运行，度量字段一致性和漏报率。
3. 调低 temperature/top-p 并固定 schema。
4. 要求关键结论绑定条款引用。
5. 建立合同样本回归集，比较参数变更前后波动。

## 核心结论
1. 采样策略决定模型如何从概率分布中选择下一个 token。
2. 低风险创意任务和高风险审计任务不应共用采样策略。
3. 稳定性要通过重复运行、schema 和引用校验来验证。
4. 采样参数不能替代检索、提示词和评测治理。
5. Agent 行动选择也需要受采样策略约束。
