# cost-routing：成本路由

## 一句话定义
成本路由是在满足质量、时延和风险底线的前提下，把请求分配给成本合适的模型或服务池。

## 为什么重要
企业 AI 成本通常被少数高频、低价值或错误路由任务放大。成本路由不是简单选择便宜模型，而是在任务价值、质量门槛、SLA、失败代价和 Token 消耗之间做策略选择。

## 心智模型
成本路由像差旅审批。普通出差坐经济舱，高风险客户现场可以升级；规则不是永远最便宜，而是让花出去的钱匹配业务价值和风险。

## 机制拆解
1. 请求先被打上任务类型、风险等级、质量要求、时延预算和预算归属。
2. 路由策略根据标签筛选候选模型或推理服务。
3. 低风险任务优先走低成本模型，高风险或失败代价高的任务走强模型。
4. 评测和线上观测持续校准哪些任务能降级、哪些必须升级。
5. 成本路由要记录 Token、模型版本、失败率和回退路径，避免只看账单。

## 企业案例
标题：客服摘要全部走旗舰模型导致成本暴涨。场景：某客服平台月 180 万次摘要请求，默认全部路由到旗舰模型。问题：两个月成本上涨 72%，但抽样评测发现 65% 低风险摘要用中模型即可达标。分析：平台没有任务分层和质量底线，只按一个默认模型服务所有请求。方案：建立摘要/投诉/合规三类任务标签，用 1 万条历史请求回放评测，低风险走中模型，高风险失败时升级旗舰。结论：成本路由要先定义质量底线，再谈省钱。工程信号：规模、指标、验证结果、约束、错误路径。

## 常见误区
1. 把成本路由等同于永远选择最便宜模型。
2. 没有评测集就直接把大比例请求切到低价模型。
3. 只看平均成本下降，不看失败率和人工返工。
4. 忽略 Token 长度，导致长输入低价值任务吞掉预算。
5. 没有回退策略，低成本模型失败后影响 SLA。

## 诊断题
场景：MaaS 平台月成本上涨 70%，团队计划把 80% 请求直接切到便宜模型，但当前没有任务分类和质量回放结果。问题：最优先的动作是什么？A. 立刻全量切换低成本模型。B. 建立任务分层和回放评测，先找出可降级且质量达标的请求。C. 只提高缓存时间，避免模型调用。D. 先购买更多低价模型额度。正确答案：B。解析：B 先确定质量底线和可降级范围；A/D 是强干扰项，会带来质量和 SLA 风险；C 只能处理重复请求，不解决模型选择。排查路径：按任务和 Token 消耗拆账单；建立回放集；定义质量底线；灰度降级；监控失败率、成本和人工返工。

## 核心结论
1. 成本路由的前提是质量和 SLA 底线。
2. 可降级任务必须由评测和线上观测证明。
3. 账单要按任务、模型、Token 和团队拆分。
4. 成本优化必须保留升级和回退路径。
