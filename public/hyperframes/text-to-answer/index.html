一句话如何变成模型回答

从自然语言到下一个 Token，模型回答是一条连续的工程链路。

Token -> Embedding -> Transformer -> 下一个 Token
01 / Token
Token 不是字数

模型先看到的不是句子，而是一串格子。

同一句话会被切成宽窄不一的 Token；上下文、成本和时延都从这些格子开始计算。

input_text
把这份合同的关键条款讲清楚

窗口占用
02 / 语义空间
编号 -> 向量 -> 距离

Token 会变成可计算的坐标。

模型处理的是向量表示；相似文本在空间里更近，但业务正确性仍要靠过滤、重排和评测。

#312 #88 #1490 #203
相似度 语义方向 合同条款 审批规则 相似但不等价
03 / Transformer
多层加工厂

Token 加上位置，再被多层结构加工。

每一层重新评估上下文关系，把局部线索变成更适合预测的内部状态。

04 / 注意力 + 位置
权重不是意识

当前位置会对历史 Token 分配权重。

线越粗，说明当前预测越依赖那个位置；顺序和距离会影响模型如何使用上下文。

05 / 自回归生成
一次一个 Token

回答不是一次写完，而是逐步接出来。

每个新 Token 都会加入上下文，成为下一步预测的条件；越长，Decode 循环越多。

合同 关键 条款 ↺
首先 关注 付款 周期 。

每一步：当前上下文 -> 概率分布 -> 选中 Token -> 接回上下文

06 / 采样策略
稳定 / 多样 / 风险

模型给概率，策略决定选哪个。

低风险任务偏稳定；创意任务可放宽随机性。企业系统要把参数和场景绑定。
