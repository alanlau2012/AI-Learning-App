02 / Prefill 与 Decode 的分工
首字前写入，首字后复用

KV Cache 不是最后一步，而是边算边积累。

Prefill 一次性处理全部输入并写入历史 K/V；Decode 每生成一个新 Token，只追加新状态并复用已有缓存。

TTFT 主要受 Prefill 长度
TPOT 主要看 Decode 每步
缓存收益来自 复用历史 K/V

Prefill: 全量输入

Decode: 一次一个

首先 关注 付款 周期

工程排查要拆开首字前和首字后：输入长不一定影响每步 TPOT，输出长一定增加 Decode 循环次数。

结论
模型回答是一条连续工程链路，不是一瞬间的理解。
01自然语言先被切成 Token，并映射为编号与向量。
02Prefill 中 Self-Attention 处理全量输入，并写入 KV Cache。
03首 Token 由概率分布和采样策略选出。
04Decode 循环复用缓存、追加新 Token，直到形成回答。
