# 12 讲课程样板内容计划

## 目标与边界

按已确认路径输出到 `[content/drafts/](content/drafts/)`，每讲一个 kebab-case JSON 文件，并生成 `[content/drafts/README.md](content/drafts/README.md)`。这些文件是 isolated draft，不参与构建、不直接作为 `src/data/*` 权威数据。

不修改 `[src/](src/)`、`[docs/content-schema.md](docs/content-schema.md)`、`[docs/animation-spec.md](docs/animation-spec.md)`、`package.json` 或任何 UI 代码。

## 内容结构

每个 JSON 草稿使用 `[docs/content-schema.md](docs/content-schema.md)` 的 `KnowledgePoint` 字段名：`id`、`title`、`slug`、`moduleId`、`order`、`difficulty`、`estimatedMinutes`、`tags`、`contentStatus`、`hasAnimation`、`definition`、`whyItMatters`、`mentalModel`、`mechanism`、`animation`、`enterpriseCase`、`pitfalls`、`diagnosticQuestion`、`keyTakeaways`、`relatedConceptIds`。

其中写作模板项会映射为权威字段：一句话定义写入 `definition`，动画脚本写入 `animation.title` 与 `animation.steps`，常见误区写入 `pitfalls`，关联知识点写入 `relatedConceptIds`。

注意：`注意力机制` 的权威 id/slug 是 `attention`，不是 `attention-mechanism`，以登记表为准。

## 动画类型

按 `[docs/animation-spec.md](docs/animation-spec.md)` 使用已定义枚举：`token-flow`、`attention-map`、`prefill-decode`、`kv-cache`、`model-router`、`context-window`、`agent-loop`、`skill-lifecycle`、`issue-fix-flow`。

`skill` 在登记表中 `hasAnimation` 当前为 false，但用户明确要求提供 `skill-lifecycle` 脚本；草稿会在 README 中标注“需审核：是否更新登记表/registry 后再入库”。

## 自检与 README

`[content/drafts/README.md](content/drafts/README.md)` 将包含：文件列表、schema 字段覆盖情况、动画脚本/诊断题/企业案例覆盖情况、最需要人工审核的 5 个点、后续内容审核 Agent 检查重点。

## 后续入库

审核通过后，由主开发按流水线将 reviewed 内容合入 `src/data/concepts.ts`，并运行 `validate:structure`、`validate:published-content`，P3 后再运行 `validate:animation`。
